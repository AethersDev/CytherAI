ARCHITECTURE OVERVIEW

Progressive Disclosure

CytherAI is designed from first principles for verifiability, reproducibility, and sovereign operation.

Deployment Model ↓

On-Premises • Air-Gapped • Sovereign

CytherAI runs entirely within your infrastructure. No cloud dependencies, no external API calls, no telemetry.

  • Fully offline-capable after initial deployment
  • Air-gap compatible for maximum security
  • Standard container deployment (Docker/Kubernetes)
  • Hardware requirements: 16GB RAM minimum, GPU optional
Threat Model ↓

Security Posture & Adversary Model

We assume adversaries with significant resources and consider the following threat vectors:

  • Supply chain compromise (mitigated via reproducible builds)
  • Runtime tampering (mitigated via integrity checks)
  • Model poisoning (mitigated via deterministic training)
  • Side-channel attacks (acknowledged, monitoring ongoing)

Current Status: Active development. External audit pending Q2 2026.

Guarantees ↓

What We Claim

  • Zero External Dependencies: No internet required post-deployment
  • Deterministic Builds: Same source + same seed = identical model
  • Reproducible Training: Full audit trail from initialization to convergence
  • Inspectable Architecture: No black boxes—every component is documented

Verification: Build hash and training logs available upon NDA.

Limitations ↓

What We Won't Claim

  • Not Production-Ready: Alpha stage. Expect bugs and API changes.
  • Not Formally Verified: Theorem proving is sound, but tooling is experimental.
  • Not Cryptographically Proven: No ZK proofs of correct execution (yet).
  • Not Immune to Hardware Attacks: Physical access = game over.
  • Performance Trade-offs: Verification adds latency. Not optimized for speed.

Timeline: Production-ready status estimated Q4 2026.
